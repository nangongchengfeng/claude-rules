# 前端设计规则

## 全局原则

### 反模板本能
- [必须] 每个页面必须有自己的视觉签名，禁止"像其他网站"
- [必须] 如果 AI 生成的页面看起来像 Vercel 模板，重做
- [优先] 用 3 秒测试：截屏后遮住 logo，还能认出是你的产品吗？

### 视觉温度
- [必须] 纯平背景色必须搭配纹理、渐变或光晕——不允许死板的纯色块
- [优先] 背景层次至少 2 层：基础色 + 纹理/渐变 + 可选光晕
- [默认] 纹理优先用 CSS 实现（噪点滤镜、网格背景），不用图片

### 像素级呼吸感
- [必须] 组件之间必须有留白，禁止内容贴边
- [优先] 间距用 4px 基数倍（4/8/12/16/24/32/48/64）
- [优先] 垂直间距大于水平间距，形成阅读节奏

### 信息层级
- [必须] 每个页面必须有且只有一个视觉焦点
- [必须] 用大小/粗细/颜色三维度拉开标题层级（不能只靠颜色）
- [优先] 主标题 → 副标题 → 正文 → 辅助文字，4 级足够

### 动画有理由
- [必须] 每个动画必须回答"为什么动"，纯装饰性动画不超过 2 个/页面
- [必须] 入场动画不超过 0.6s，交互反馈不超过 0.3s
- [优先] 优先用 spring 物理动画或 ease-out 系曲线
- [必须] 禁止线性动画（linear），除非模拟匀速物理运动（如进度条）

## 配色框架

### 色彩角色
每个项目必须定义以下 5 个色彩角色，具体色值由项目定制：

| 角色 | 用途 | 示例方向 |
|------|------|----------|
| 主色（Primary） | 品牌、CTA、关键操作 | 暖色系优先，避开蓝紫 |
| 表面色（Surface） | 背景、卡片底色 | 深色/浅色各一套 |
| 前景色（Foreground） | 正文、标题 | 与表面对比度 ≥ 4.5:1 |
| 强调色（Accent） | 链接、标签、图标高亮 | 与主色互补或对比 |
| 语义色（Semantic） | 成功/警告/错误/信息 | 保持行业惯例直觉 |

### 禁忌色
- [必须] 禁止 #6366F1（靛蓝）、#8B5CF6（紫色）作为主色或强调色
- [必须] 禁止蓝紫渐变作为背景或按钮
- [优先] 蓝色系主色必须偏青或偏钢（如 #0EA5E9、#3B82F6 需搭配暖色平衡）
- [默认] 灰色系（slate/zinc/neutral）作为中性色不受限制

### 深色模式
- [必须] 深色模式下表面色不得纯黑（#000），至少 #0A0A0A 起
- [优先] 深色模式前景色不用纯白（#FFF），用 rgba(255,255,255,0.87) 或 #F5F5F5
- [优先] 深色模式强调色饱和度比浅色模式提高 15-20%

### 渐变
- [优先] 渐变必须自然，禁止 90° 硬切渐变
- [优先] 渐变角度偏向斜角（135°/225°），避免纯水平/垂直
- [优先] 渐变端点色相差异不超过 60°（除非刻意追求撞色效果）

### 对比度
- [必须] 正文与背景对比度 ≥ 4.5:1（WCAG AA）
- [必须] 大标题与背景对比度 ≥ 3:1
- [优先] 关键操作按钮与背景对比度 ≥ 7:1

## 展示型规则

适用于：落地页、营销页、产品首页、个人主页等以视觉冲击力为核心的页面。

### 布局
- [必须] Hero 区域禁止"标题居中 + 副标题 + 按钮"的经典三件套
- [必须] 页面至少一处使用不对称布局（左右比例 6:4 或 7:3）
- [优先] 内容块之间用视觉重量变化制造节奏感，不是等高堆叠
- [优先] 关键数据/卖点用超大字号呈现（80px+），形成视觉锚点
- [默认] 折叠/滚动触发内容用错位入场，不是同步淡入

### 视觉签名
- [必须] 每个展示型页面必须有 1 个"签名动效"——一眼记住的那种
  - 例：3D 倾斜卡片、粒子背景、文字逐字显现、鼠标跟随光效
- [优先] 签名动效放在首屏，3 秒内出现
- [优先] 签名动效用 CSS/Framer Motion 实现，不依赖重框架

### 组件深度定制
- [必须] 使用 UI 组件库时必须覆盖默认样式，不得直接使用开箱即用外观
- [优先] 按钮必须有悬停态（hover）和按下态（active），至少两个状态变化
- [优先] 卡片组件至少定制：圆角、阴影、边框或光效之一
- [默认] 表单输入框聚焦态必须有明显视觉反馈（光晕/边框色变化）

### 首屏体验
- [必须] 首屏加载 1.5s 内必须有意义内容出现（骨架屏或渐进式渲染）
- [优先] 首屏动画序列：签名动效 → 标题入场 → CTA 出现，间隔 ≤ 0.3s
- [优先] CTA 按钮必须在首屏可见，不需要滚动
- [默认] 背景光效/纹理在 HTML 加载后即开始，不等 JS

### 滚动体验
- [必须] 长页面至少 3 处滚动触发动画（Intersection Observer）
- [优先] 滚动动画用 stagger（交错）而非同步出现
- [优先] 视差效果用于不超过 2 个层次，避免晕眩
- [默认] 回到顶部按钮在滚动超过 2 屏后出现

## 后台管理规则

适用于：SaaS 后台、仪表盘、管理系统、内部工具等以功能和效率为核心的页面。

### 布局
- [必须] 导航结构在 1 秒内让用户知道"我在哪、能去哪"
- [优先] 侧边栏导航宽度 240-280px，折叠态 64px
- [优先] 内容区最小宽度 1024px，支持宽屏利用
- [默认] 表格区域占据主内容区 70%+ 宽度
- [默认] 统计卡片允许等宽排列（与展示型不同），但必须用视觉层级区分主次

### 信息密度
- [必须] 后台页面每屏信息密度高于展示型页面，但不拥挤
- [优先] 表格行高 48-56px，单元格内边距 12-16px
- [优先] 关键操作按钮固定在视口（sticky header 或底部操作栏）
- [默认] 列表页每屏展示 15-25 条数据，配合虚拟滚动

### 表单
- [必须] 表单提交必须有加载态（按钮 disabled + spinner），防止重复提交
- [必须] 表单校验即时反馈，不等到提交时才报错
- [优先] 长表单分步骤（stepper），每步不超过 7 个字段
- [优先] 日期/时间选择器必须支持键盘输入，不只依赖弹窗
- [默认] 下拉选项超过 10 个时使用搜索选择器

### 数据展示
- [必须] 数字型数据必须格式化（千分位、单位、精度），不得裸露原始数值
- [优先] 统计卡片包含：标题 + 主数值 + 变化趋势（↑↓ + 百分比）
- [优先] 表格支持列排序和筛选，默认按时间倒序
- [默认] 空状态有插图或引导文案，不是白板

### 导航与搜索
- [必须] 全局搜索入口在顶部导航栏，快捷键 Ctrl/Cmd+K
- [优先] 面包屑导航在内容区顶部，支持点击跳转
- [优先] 最近访问/常用功能快捷入口可见
- [默认] 侧边栏当前项高亮，父级展开

### 后台不丑
- [优先] 后台也要有品牌色——用主色点缀导航高亮、按钮、标签
- [优先] 表格交替行用极浅色区分（rgba 透明度 0.02-0.04）
- [优先] 页面切换用淡入过渡，不闪切
- [默认] 空白页用品牌色系插画，不是灰色方块

## 动画体系

### 缓动曲线
- [必须] 禁止 linear 曲线（匀速），除非模拟进度条等匀速物理运动
- [必须] 入场动画优先使用 ease-out 系曲线，退场用 ease-in 系
- [优先] 推荐曲线库：
  - 入场：`cubic-bezier(0.16, 1, 0.3, 1)`（ease-out-expo，快速到位轻柔停止）
  - 退场：`cubic-bezier(0.7, 0, 0.84, 0)`（ease-in-expo，缓慢加速离场）
  - 弹性：spring(tension=170, friction=26)（Framer Motion 默认 spring）
  - 微交互：`cubic-bezier(0.34, 1.56, 0.64, 1)`（overshoot，轻弹回感）
- [默认] 不知道用什么时，用 ease-out-expo

### 时长
- [必须] 入场动画 ≤ 600ms，交互反馈 ≤ 300ms，退场 ≤ 400ms
- [优先] hover/focus 状态变化 150-200ms
- [优先] 页面切换过渡 300-500ms
- [默认] 超过 600ms 的动画必须拆分为多段

### 交错（Stagger）
- [必须] 列表/卡片组入场使用交错动画，禁止同步出现
- [优先] 交错间隔 50-100ms，元素超过 10 个时用递减间隔
- [优先] 交错方向与阅读方向一致（中文/英文：从左到右，从上到下）
- [默认] 交错动画总时长不超过 1.5s

### 微交互
- [必须] 按钮 hover 有至少 1 个视觉变化（色变/缩放/阴影/位移）
- [优先] 按钮 active 按下缩放 scale(0.97)，松开回弹
- [优先] 复选框/开关切换有滑动或弹跳感
- [优先] 数字变化用计数动画（countUp），不跳变
- [默认] tooltip 出现延迟 300ms，消失延迟 100ms

### 性能
- [必须] 动画只用 transform 和 opacity，不触发 layout/paint
- [必须] will-change 只在动画期间添加，动画结束移除
- [优先] 复杂动画用 CSS @keyframes 优先于 JS，减少主线程负担
- [优先] 减弱动画偏好 prefers-reduced-motion 必须尊重，降级为即时切换

## 文案风格

### 语气
- [必须] 口语化，像跟朋友聊天，不像写论文
- [必须] 具体化，有数字和场景，不说空话
- [优先] 可以幽默、自嘲、甚至挑衅，但不超过 1 处/页面
- [默认] 每句话尽量控制在 15 字以内，长句拆成短句

### 禁止
- [必须] 禁止 Lorem Ipsum 占位文本，AI 生成时必须写有意义的文案
- [必须] 禁止被动语态（"被用于"→"用于"）
- [必须] 禁止无意义的行业黑话（"赋能"、"生态"、"闭环"）
- [优先] 禁止超长复合句（一个句子 3 个以上逗号就太长了）

### 标题与按钮
- [必须] 标题说结果/利益，不说功能
  - ✗ "实时监控引擎" → ✓ "3 秒发现异常"
  - ✗ "智能推荐系统" → ✓ "比你更懂你想要什么"
- [必须] 按钮文案说动作，不说状态
  - ✗ "提交成功" → ✓ "提交"（成功用 toast 反馈）
  - ✗ "正在加载" → ✓ "加载"（进度用 spinner）
- [优先] CTA 按钮用动词开头：开始/创建/获取/探索

### 辅助文案
- [必须] 空状态必须有引导文案和下一步操作入口
- [优先] 错误信息说"怎么办"，不只说"出错了"
  - ✗ "网络错误" → ✓ "网络不给力，点击重试"
- [优先] 加载状态有进度暗示，不只是 spinner
  - ✗ 转圈 → ✓ "正在分析数据..." / 进度条
- [默认] placeholder 文案用示例而非说明
  - ✗ "请输入邮箱" → ✓ "you@example.com"

## 图片与图标系统

### 图标
- [必须] 功能图标使用 Iconify 图标库（https://iconify.design）
- [必须] 图标风格统一：同一项目内只用一套图标集（推荐 Phosphor/Lucide/Heroicons）
- [优先] 图标大小与行高对齐：16px 图标配 16px 行高文字，20px 图标配 20px
- [优先] 可点击图标最小触控区域 44×44px（含 padding）
- [必须] 禁止用 Emoji 替代功能图标

### 图片
- [必须] 占位图使用 Picsum Photos（https://picsum.photos），不用灰色方块
- [优先] 真实产品图使用 Pexels（https://www.pexels.com）搜索
- [优先] 插画使用 unDraw（https://undraw.co），配色可定制
- [必须] 所有图片必须设 width/height 或 aspect-ratio，防止布局偏移（CLS）
- [优先] 图片懒加载：首屏以下用 loading="lazy"

### 图标与文字搭配
- [必须] 图标 + 文字组合时，图标在文字左侧（RTL 布局除外）
- [优先] 纯图标按钮必须有 aria-label 或 title 属性
- [优先] 导航图标和文字同时存在时，图标仅作视觉辅助，不承载唯一语义

## UI 组件库选择指南

### 选择策略
- [必须] 项目启动时必须明确选择一个主 UI 组件库，不混用
- [优先] 根据项目类型选择：

| 项目类型 | 推荐组件库 | 理由 |
|----------|-----------|------|
| 展示型落地页 | Aceternity UI / Magic UI | 动效丰富，视觉冲击力强 |
| 后台管理系统 | Ant Design / Mantine | 表单/表格/导航功能完善 |
| 赛博朋克/游戏风 | DaisyUI (cyberpunk 主题) | 开箱即用的主题风格 |
| 极简/编辑器类 | Radix UI + 手写样式 | 无样式原语，完全自定义 |
| Svelte 项目 | ikun-ui | Svelte 生态原生支持 |

### 定制要求
- [必须] 使用任何组件库都必须覆盖主题变量，不得使用开箱即用默认外观
- [优先] 展示型项目：至少定制 3 个核心组件的默认样式
- [优先] 后台管理项目：通过主题 token 覆盖配色和圆角，不逐组件覆盖
- [默认] 组件库的默认动画可以用，但入场动画必须替换为自定义缓动

### 禁止
- [必须] 禁止同时引入 2 个以上 UI 组件库（样式冲突 + 包体积）
- [必须] 禁止直接复制 Shadcn/Material UI 组件代码而不做任何视觉定制
- [优先] 避免使用组件库中视觉辨识度极高的默认样式（一看就知道用的什么库）

## 排版与字体

### 字体选择
- [必须] 每个项目必须定义正文字体和标题字体（可以相同）
- [优先] 西文标题推荐：Inter / Plus Jakarta Sans / Space Grotesk
- [优先] 中文正文推荐：系统字体栈（-apple-system, "PingFang SC", "Microsoft YaHei"）
- [优先] 等宽字体推荐：JetBrains Mono / Fira Code（代码块和数字展示）
- [必须] 字体栈必须包含系统回退字体，不依赖外部字体加载

### 字号层级
- [必须] 页面字号层级不超过 5 级：
  - L1 主标题：32-64px（展示型可到 80px+），font-weight 700-900
  - L2 副标题：20-28px，font-weight 600-700
  - L3 小标题：16-18px，font-weight 600
  - L4 正文：14-16px，font-weight 400
  - L5 辅助文字：12-13px，font-weight 400，opacity 0.6
- [优先] 展示型项目 L1 可以更极端（96px+），后台管理 L1 不超过 32px
- [必须] 相邻层级字号比至少 1.2 倍，拉开视觉差距

### 行高与间距
- [必须] 正文行高 1.5-1.7，标题行高 1.1-1.3
- [优先] 段落间距 ≥ 1.5 倍行高
- [优先] 标题与正文间距 > 正文行间距（制造节奏感）
- [默认] 长文本最大宽度 680px，避免单行过长

### 字重与颜色
- [优先] 标题和正文用字重拉开层级，不是靠颜色
- [必须] 正文颜色不纯黑，用 rgba(0,0,0,0.87) 或 #1A1A1A
- [优先] 辅助文字比正文浅 1-2 档，不是用另一个颜色
- [默认] 深色模式正文用 rgba(255,255,255,0.87)，辅助 rgba(255,255,255,0.6)
